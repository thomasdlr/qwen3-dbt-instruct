Generate dbt fine-tuning dataset using Claude Sonnet 4.6

Task: Read each query from data/selected_queries.jsonl and generate production-ready multi-file dbt DAGs. Append results to data/dbt_dag_dataset_sonnet.jsonl. Each output row must have all original fields plus dbt_dag (string of XML-tagged files) and status set to success.

Output format: Each dbt_dag string contains XML file tags for:
- models/staging/stg_TABLENAME.sql (one per source table)
- models/staging/_staging_sources.yml
- models/staging/_staging_models.yml
- models/marts/core/fct_or_dim_or_rpt_NAME.sql
- models/marts/core/_core_models.yml

Use source('raw','table') in staging, ref('stg_...') in marts, config(materialized=...) in all files, snake_case columns, never SELECT *. See CLAUDE.md for full conventions.

Process rows in batches. Skip any row whose question already exists in the output file. Do NOT create intermediate JSON files. Go directly from SQL to dbt XML output.

When done, run these verification commands and print their output:
