# Qwen3-4B-DBT-Instruct

Fine-tuning pipeline that teaches a small language model to turn natural language business questions into complete, production-ready **dbt DAGs** (staging SQL + intermediate models + marts + YAML schema files).

---

## Overview

| | |
|---|---|
| **Base model** | [`unsloth/Qwen3-4B-Instruct-2507-unsloth-bnb-4bit`](https://huggingface.co/unsloth/Qwen3-4B-Instruct-2507-unsloth-bnb-4bit) |
| **Training method** | QLoRA (rank 32, alpha 32) via [Unsloth](https://github.com/unslothai/unsloth) + TRL SFTTrainer |
| **Dataset** | 1 000 synthetic SQL → dbt DAG pairs, generated by Claude Sonnet and validated with `dbt parse` |
| **Training hardware** | Google Colab free tier (T4, 15 GB VRAM) |
| **LoRA adapter** | [`tdelard/Qwen3-4B-DBT-Instruct-LoRA`](https://huggingface.co/tdelard/Qwen3-4B-DBT-Instruct-LoRA) |
| **Quantised model** | [`tdelard/Qwen3-4b-DBT-Instruct-GGUF`](https://huggingface.co/tdelard/Qwen3-4b-DBT-Instruct-GGUF) (Q4_K_M, ~2.7 GB) |
| **Dataset on HF** | [`tdelard/text_to_dbt`](https://huggingface.co/datasets/tdelard/text_to_dbt) |

The fine-tuned model generates the full multi-file dbt project structure from a single question — staging models with `source()` references, intermediate transformations, mart models, and `sources.yml` / schema YAML — all correctly formatted and parseable by `dbt parse`.

## Phase 1 — Data generation (`01_data_generation/`)

Builds the fine-tuning dataset from scratch using a local LLM (LM Studio) to transform SQL queries into dbt DAGs. Validation is done with an actual `dbt-duckdb` project so only parseable examples enter the dataset.

```
b-mc2/sql-create-context (HuggingFace)
        │
        ▼
select_queries.py          # Complexity scoring + stratified sampling → 1 000 queries
        │
        ▼
transform_sql_to_dbt.py    # LLM call (Claude Sonnet) → multi-file dbt DAG per query
        │
        ▼
validate_dbt_dag.py        # dbt parse + structural checks → removes broken examples
        │
        ▼
data/validated_dataset_v2.jsonl   (~950 clean rows)
```

### Quick start

```bash
cd 01_data_generation
uv sync
cp .env.example .env     # set ANTHROPIC_API_KEY (or LOCAL_LM_BASE_URL for offline)
uv run select_queries.py --target 1000 --output data/selected_queries.jsonl
uv run transform_sql_to_dbt_sonnet.py --output data/dbt_dag_dataset.jsonl
uv run validate_dbt_dag.py --input data/dbt_dag_dataset.jsonl --output data/validated_dataset.jsonl
```

## Phase 2 — Fine-tuning (`02_finetuning/`)

QLoRA training on Google Colab. The notebook handles everything: base model loading, LoRA adapter setup, training, GGUF export, and push to HuggingFace.

### Training configuration

| Parameter | Value |
|---|---|
| Base model | `unsloth/Qwen3-4B-Instruct-2507-unsloth-bnb-4bit` |
| LoRA rank / alpha | 32 / 32 |
| Learning rate | 2e-4 |
| Max steps | 60 |
| Max sequence length | 2 048 |
| Loss masking | Response tokens only |
| Batch size | 2 (grad. accum. 4) |
| Optimizer | adamw_8bit |

### Files

| File | Purpose |
|---|---|
| `prepare_dataset.py` | Convert validated dataset → chat format, 90/10 train/eval split |
| `qwen3_dbt_finetune.ipynb` | Colab training notebook (Unsloth + TRL SFTTrainer + GGUF export) |
| `evaluate_baseline.py` | Evaluate any LM Studio model on the eval set |
| `data/train.jsonl` | 900-row training set |
| `data/eval.jsonl` | 100-row eval set |
| `results/baseline.json` | Baseline metrics (pre-fine-tuning) |

### Quick start

**1. Prepare the dataset** (already done — artefacts committed):
```bash
cd 02_finetuning
python prepare_dataset.py
```

**2. Run the Colab notebook** — open `qwen3_dbt_finetune.ipynb` in Google Colab, enable T4 GPU, run all cells. It will push the LoRA adapter and the GGUF file to HuggingFace automatically.

**3. Baseline evaluation** (optional — requires LM Studio running locally):
```bash
python evaluate_baseline.py --model "qwen3-4b" --limit 50
```

### Baseline results (pre-fine-tuning, Ministral-8B)

| Metric | Score |
|---|---|
| `dbt parse` pass rate | 51% |
| Has staging layer | 98% |
| Has marts layer | 100% |
| Has `sources.yml` | 100% |
| Has `ref()` calls | 98% |
| Avg files generated | 6.25 |

> The 51% `dbt parse` pass rate on the untuned model shows why fine-tuning is needed: the model produces structurally plausible but syntactically broken DAGs half the time.

## Inference

### With the GGUF (LM Studio / llama.cpp)

Download [`tdelard/Qwen3-4b-DBT-Instruct-GGUF`](https://huggingface.co/tdelard/Qwen3-4b-DBT-Instruct-GGUF), load in LM Studio, and send requests with the system prompt:

```
You are a dbt expert. Given a business question and SQL context, generate a complete dbt project DAG with staging models, intermediate models, marts, and YAML schema files.
```

### With the LoRA adapter (Python)

```python
from transformers import AutoTokenizer, AutoModelForCausalLM
from peft import PeftModel

base_model_id = "unsloth/Qwen3-4B-Instruct-2507-unsloth-bnb-4bit"
adapter_id    = "tdelard/Qwen3-4B-DBT-Instruct-LoRA"

tokenizer = AutoTokenizer.from_pretrained(base_model_id)
model     = AutoModelForCausalLM.from_pretrained(base_model_id, torch_dtype="auto", device_map="auto", load_in_4bit=True)
model     = PeftModel.from_pretrained(model, adapter_id)
```
